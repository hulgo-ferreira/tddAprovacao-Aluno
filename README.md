# TDD - Regra de Aprovação de Aluno

Este projeto demonstra a aplicação de TDD (Test Driven Development) na implementação de uma regra de negócio simples: determinar se um aluno está aprovado ou reprovado com base em critérios de frequência e notas.

## 📌 Regras de Negócio

- Frequência mínima: 75%
- Nota final mínima: 60
- Caso não atinja a nota mínima, é possível aprovação por média entre nota final e nota especial

## 🧪 Estratégia de Testes

Foram aplicados testes unitários utilizando:
- Casos positivos e negativos
- Testes de borda (boundary values)
- Validação de cenários críticos

## 🎯 Objetivos

- Demonstrar evolução de código guiada por testes (TDD)
- Aplicar boas práticas de escrita de testes
- Garantir legibilidade e simplicidade no design

## 🛠️ Tecnologias

- Java
- JUnit 5
- Maven
